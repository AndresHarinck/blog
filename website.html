Hoe mijn website tot stand kwam

Een blik achter de schermen van mijn e‑portfolio met technologie, structuur en een beetje AI magie

1. Opbouw van de website

Mijn e‑portfolio is volledig gebouwd met HTML en CSS. Geen WordPress, geen templates en geen frameworks. Gewoon zelf schrijven en alles onder controle houden.

Ik heb geprobeerd om de HTML en CSS vaardigheden en de best practices toe te passen die ik in het eerste jaar heb geleerd. Het was leuk om dat opnieuw te gebruiken en te merken dat het nog goed in mijn vingers zit. Ik moet wel eerlijk zijn. Ik heb veel div elementen gebruikt. Dimitri Casier zal daar waarschijnlijk niet enthousiast van worden, maar voor mij werkte het snel en overzichtelijk. Het maakte het stylen eenvoudiger en zorgde ervoor dat ik vlot kon doorwerken.

Ik heb ook AI gebruikt om bepaalde CSS problemen sneller op te lossen. Niet omdat ik het niet kon, maar omdat ik geen zin had om drie uur te zoeken naar een fout die uiteindelijk veroorzaakt werd door een spatie die verkeerd stond. AI was in dat geval een ideale collega die nooit moe wordt en altijd klaarstaat om te helpen.

2. Technologieën die ik gebruikt heb

De website draait volledig op:

  • HTML voor de structuur
  • CSS voor de layout en stijlen
  • GitHub Pages voor de hosting
  • GitHub voor versiebeheer

Alles is handgeschreven, maar soms wel met AI als CSS hulplijn wanneer ik er zelf genoeg van had.

3. Hosting via GitHub Pages

De website wordt gehost via GitHub Pages. Zodra ik een wijziging push, wordt de site automatisch opnieuw gepubliceerd. Geen gedoe met servers of configuratie. Gewoon committen, pushen en klaar.

4. Hoe ik nieuwe content toevoeg

Een nieuwe pagina toevoegen gaat snel en eenvoudig.

  1. Een nieuw HTML bestand maken
  2. De basisstructuur kopiëren van een bestaande pagina
  3. Tekst en foto’s invullen
  4. Committen en pushen naar GitHub
  5. GitHub Pages publiceert de update automatisch

Gemiddeld kost het mij vijf tot tien minuten om een nieuwe post toe te voegen.

5. Wat ik geleerd heb

Door deze website te bouwen heb ik mijn HTML en CSS vaardigheden versterkt. Ik heb opnieuw ervaren hoe handig versiebeheer is en hoe eenvoudig GitHub Pages werkt. Ik heb ook geleerd dat AI een ideale partner is wanneer ik vastloop op kleine details.
